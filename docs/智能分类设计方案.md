# 智能分类 (v0.4) — 设计方案

## 目标

每笔交易自动打上标签，支持人工修正。本地规则优先，AI 兜底。

## 分类流程

```
交易创建（手动/导入）
        ↓
1. 本地规则匹配：同商户名 → 用户历史标签
        ↓ 未命中
2. AI 兜底：DeepSeek 根据商户名+描述推断标签
   约束：优先从预设标签库选择，无匹配时生成长度≤4字新标签
        ↓
3. 自动写入 tags，用户可随时修改
```

## 后端改动

### TransactionResponse 加入 tags

```python
class TransactionResponse(BaseModel):
    # ... existing fields ...
    tags: list[TagResponse] = []
```

查询时 eager-load `Transaction.tags` → `TransactionTag.tag`。

### Tag DELETE 端点

`DELETE /tags/{tag_id}` — 仅限用户自定义标签。删除时清理 TransactionTag 关联。

### 导入后自动分类

`import_pipeline.py` 的 `_run_import` 末尾，对所有新建交易调用 AI 分类：

```python
from app.services.ai_classification import classify_transaction

for new_tx in created_transactions:
    result = await classify_transaction(db, user_id, new_tx)
    # result → [{tag_id, name}] → create TransactionTag rows
```

### 本地规则存储

已有 `ai_classification.py` 实现了：同一 `merchant_name` → 查找历史标签。不需要新增表。

## 前端改动

### TransactionRow — 标签显示

在类型徽章后面显示标签胶囊：

```tsx
{tx.tags?.map(tag => (
  <span className="text-xs bg-gray-100 text-gray-600 px-1.5 py-0.5 rounded">
    {tag.emoji} {tag.name}
  </span>
))}
```

### TransactionFilters — 标签筛选

增加标签下拉选择器，列出所有可用标签。已选中标签高亮。

### TransactionEditor — 集成标签选择

在编辑器中新增"标签"区域，复用 TagEditor 的选择逻辑（多选药丸按钮）。

### 标签管理页面

路径 `/tags`，导航栏添加入口。

- 标签列表（预设 + 自定义），分组显示
- 新建：名称 + emoji + 父标签
- 编辑：修改名称/emoji
- 删除：仅限自定义标签，确认后删除
