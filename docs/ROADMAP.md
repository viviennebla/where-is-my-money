# WIMM Roadmap

## v0.1 — MVP 核心 (已完成)

**目标：** 跑通"文件上传 → AI 解析 → 去重落库"主线链路。

- [x] JWT 用户认证
- [x] 账户 CRUD（银行卡、软件余额、理财、月付、分期）
- [x] 交易 CRUD（支出、收入、转账、退款），余额联动
- [x] 导入流水线：上传 → AI 推断列映射 → 人工确认 → 账户绑定 → 执行
- [x] 合并引擎：external_tx_id 强匹配 + amount+type+24h 时间窗模糊匹配
- [x] 文件解析器：CSV + Excel，自动检测标题行
- [x] DeepSeek API 集成（列推断 + 账户匹配）
- [x] 导入模板保存与复用
- [x] 前端：流水列表、账户管理、导入向导（5 步）

---

## v0.2 — 反馈修复 (已完成)

**目标：** 补齐核心体验缺口，修复数据质量。

- [x] 交易列表内联编辑（点击字段即编辑）
- [x] 转账支持（转出 + 转入账户）
- [x] 交易列表显示账户名
- [x] 账户详情页：最新交易在前 + 实时余额
- [x] 余额调整（BALANCE_ADJUSTMENT 类型，含日期和备注）
- [x] 银行卡账户：别名 + 卡号
- [x] 退款标记与关联（parent_id）
- [x] AI 退款智能匹配（商户名模糊匹配，策略 1-3）
- [x] 微信预设模板
- [x] 52 个单元测试

---

## v0.3 — 数据质量与体验 (已完成)

**目标：** 模板可迭代、操作可视化、余额可靠。

- [x] 模板编辑器：查看/修改映射 → 保存并更新历史数据 或 另存为新模板
- [x] 统一异步任务 + 实时进度条（导入 + 重处理）
- [x] 导入流水线自动更新账户余额
- [x] 转入交易纳入余额计算（recalculate + running balance）
- [x] 微信交易类型拼接到商户名（"商户消费 · 美团"）
- [x] 交易列表分页（每页 20/50/100）
- [x] 筛选条件切换自动回到第 1 页

---

## v0.4 — 智能分类 (已完成)

**目标：** AI 自动给交易打标签，用户可管理分类体系。

- [x] 标签 CRUD + 标签树（预设 + 自定义）
- [x] AI 智能分类：DeepSeek 根据商户名/描述推断标签，强制优先从预设标签库选择
- [x] 本地规则优先：同商户名 → 查找历史标签
- [x] 交易编辑器中快速选择/修改标签
- [x] 标签筛选流水
- [x] 标签管理页面（创建、编辑、删除）
- [x] 导入后自动分类（新交易自动打标签）
- [x] 修复 ai_classification.py：复用 _call_deepseek()，修复 system role、超时、token 限制

---

## v0.5 — 数据看板 (已完成)

**目标：** 直观展示资金流向和消费结构。

- [x] 月度收支概览（收入/支出/结余趋势图）
- [x] 分类饼图/柱状图（按 source_category 统计）
- [x] 账户余额一览（比例条）
- [x] 商户排名 Top 10（水平柱状图）
- [x] 自定义时间范围筛选（年/月 + 本年/全部快捷按钮）
- [x] 统计 API：summary, monthly-trend, category-breakdown, merchant-ranking, account-balances

---

## v0.6 — 本地优先 & 离线 (规划中)

**目标：** 数据本地化，离线可用，隐私可控。

- [ ] IndexedDB (Dexie.js) 前端本地存储
- [ ] 服务端同步机制（last_sync_success 时间戳）
- [ ] 离线可浏览交易流水和账户
- [ ] 网络恢复后自动同步
- [ ] 同步状态 UI 展示（"上次同步：X 分钟前"）
- [ ] PWA 支持（Service Worker 缓存静态资源）
